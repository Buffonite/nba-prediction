How it works

Four-stage pipeline: data acquisition → feature engineering → model training → bracket simulation.

1

Data acquisition (src/data_fetch_bref.py)

Scrapes monthly schedules from Basketball Reference for 4 seasons of NBA games — regular season and playoffs, ~5,257 games total.

2

Feature engineering (src/preprocessing.py + src/elo.py)

For every game, computes each team's rolling 5- and 10-game stats (win %, points scored/allowed, net rating), rest days, and pre-game ELO ratings (FiveThirtyEight-style with home advantage and MOV). Strict .shift(1) guards against data leakage.

3

Model training (src/model.py + src/train.py)

Dense network with BatchNorm, Dropout (0.3), and early stopping on validation AUC. Logistic regression baseline trained on the same features for comparison.

4

Bracket simulation (src/playoffs.py)

Monte-Carlo simulates 10,000 full-bracket continuations. Each best-of-7 series respects the 2-2-1-1-1 home-court format and uses single-game probabilities (cached) for each potential matchup.

Honest caveats. This model is trained on regular-season games — it does not capture playoff-specific dynamics (tighter rotations, scheme adjustments, key injuries). Star-player availability is implicitly assumed at 100% in the current run. Treat the percentages as the model's view of an idealised series, not a betting recommendation.
